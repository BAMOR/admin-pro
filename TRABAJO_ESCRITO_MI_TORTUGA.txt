1. INTRODUCCIÓN
================================================================================

En la era digital actual, el comercio electrónico se ha convertido en una 
herramienta fundamental para las empresas que buscan expandir su alcance y 
ofrecer servicios de venta en línea. El presente proyecto, denominado "Mi 
Tortuga", es un sistema de comercio electrónico desarrollado con tecnologías 
web modernas que permite a los usuarios realizar compras en línea de manera 
segura y eficiente.

El sistema está diseñado para ofrecer una experiencia completa tanto para 
clientes como para administradores, incluyendo funcionalidades como gestión 
de inventario, procesamiento de pedidos, seguimiento de envíos, sistema de 
reseñas, chat en vivo y generación de reportes.

================================================================================
2. OBJETIVOS DEL PROYECTO
================================================================================

OBJETIVO GENERAL:
-----------------
Desarrollar una plataforma de comercio electrónico funcional que permita a 
los usuarios adquirir productos en línea y a los administradores gestionar 
eficientemente el inventario, ventas y clientes.

OBJETIVOS ESPECÍFICOS:
----------------------
• Implementar un sistema de autenticación seguro con roles diferenciados
  (administrador y cliente)

• Crear un catálogo de productos con filtros de búsqueda avanzados

• Desarrollar un carrito de compras con gestión de cantidades y cálculo 
  automático de totales

• Integrar un proceso de checkout con múltiples métodos de pago y envío

• Implementar un sistema de seguimiento de pedidos en tiempo real

• Desarrollar un panel administrativo para gestión de inventario, ventas 
  y usuarios

• Incorporar un sistema de reseñas y calificaciones de productos

• Implementar un chat en vivo para atención al cliente

• Generar facturas en formato PDF automáticamente

• Crear reportes de ventas con gráficos estadísticos

================================================================================
3. DESCRIPCIÓN DEL SISTEMA
================================================================================

"Mi Tortuga" es una aplicación web de comercio electrónico que conecta a 
vendedores con compradores a través de una interfaz intuitiva y moderna. 
El sistema cuenta con dos tipos de usuarios principales:

USUARIOS CLIENTES:
------------------
Los clientes pueden navegar por el catálogo de productos, realizar búsquedas 
filtradas, agregar productos al carrito, completar compras, dejar reseñas, 
chatear con soporte y dar seguimiento a sus pedidos.

USUARIOS ADMINISTRADORES:
-------------------------
Los administradores tienen acceso a un panel de control donde pueden gestionar 
el inventario de productos, procesar ventas, administrar usuarios, responder 
consultas del chat y generar reportes de rendimiento del negocio.

CARACTERÍSTICAS PRINCIPALES:
----------------------------
• Diseño responsivo con interfaz moderna y elegante
• Sistema de autenticación con encriptación de contraseñas
• Carrito de compras persistente por sesión
• Múltiples métodos de pago (tarjeta, PayPal, transferencia)
• Opciones de envío (estándar y express)
• Notificaciones de stock bajo para administradores
• Alertas visuales con SweetAlert2
• Exportación de documentos en PDF

4. ARQUITECTURA Y TECNOLOGÍAS UTILIZADAS
================================================================================

ARQUITECTURA DEL SISTEMA:
-------------------------
El proyecto sigue una arquitectura MVC (Modelo-Vista-Controlador) simplificada,
donde:

• MODELO: Representado por las clases y funciones que interactúan con la base
  de datos (config/database.php)

• VISTA: Los archivos PHP que generan el HTML enviado al navegador
  (index.php, admin.php, etc.)

• CONTROLADOR: La lógica de negocio implementada en los mismos archivos PHP
  que procesan las solicitudes del usuario

TECNOLOGÍAS DEL LADO DEL SERVIDOR:
----------------------------------
• PHP 8.2: Lenguaje principal de programación
• PDO (PHP Data Objects): Para conexión segura a base de datos
• MySQL/MariaDB: Sistema de gestión de base de datos
• TCPDF: Librería para generación de documentos PDF
• Composer: Gestor de dependencias de PHP

TECNOLOGÍAS DEL LADO DEL CLIENTE:
---------------------------------
• HTML5: Estructura semántica de las páginas
• CSS3: Estilos con diseño moderno y animaciones
• JavaScript: Interactividad del lado del cliente
• SweetAlert2: Librería para alertas elegantes
• Chart.js: Librería para gráficos estadísticos
• Google Fonts: Tipografías Playfair Display y Poppins

SERVIDOR WEB:
-------------
• Apache o Nginx (recomendado XAMPP para desarrollo local)

6. BASE DE DATOS
================================================================================

SISTEMA GESTOR: MySQL / MariaDB
NOMBRE DE LA BASE DE DATOS: sistema_compras

TABLAS IMPLEMENTADAS:
---------------------

1. TABLA: usuarios
   Propósito: Almacenar información de los usuarios del sistema
   
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - nombre (VARCHAR(100), NOT NULL)
   - email (VARCHAR(100), NOT NULL, UNIQUE)
   - password (VARCHAR(255), NOT NULL)
   - rol (ENUM('admin','cliente'), DEFAULT 'cliente')
   - creado (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

2. TABLA: productos
   Propósito: Almacenar el catálogo de productos disponibles
   
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - nombre (VARCHAR(150), NOT NULL)
   - descripcion (TEXT)
   - precio (DECIMAL(10,2), NOT NULL)
   - stock (INT, DEFAULT 0)
   - categoria (VARCHAR(100))
   - imagen (VARCHAR(255), DEFAULT 'default.png')
   - creado (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

3. TABLA: ventas
   Propósito: Registrar todas las transacciones de compra
   
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - usuario_id (INT, FOREIGN KEY REFERENCES usuarios(id))
   - total (DECIMAL(10,2), NOT NULL)
   - metodo_pago (VARCHAR(50), NOT NULL)
   - metodo_envio (VARCHAR(50), NOT NULL)
   - estado (ENUM con 5 estados, DEFAULT 'Confirmado')
   - fecha (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

4. TABLA: reseñas
   Propósito: Almacenar opiniones y calificaciones de productos
   
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - producto_id (INT, FOREIGN KEY REFERENCES productos(id))
   - usuario_id (INT, FOREIGN KEY REFERENCES usuarios(id))
   - comentario (TEXT, NOT NULL)
   - calificacion (INT, CHECK entre 1 y 5)
   - fecha (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

5. TABLA: chat
   Propósito: Guardar mensajes del chat de soporte
   
   Campos:
   - id (INT, PRIMARY KEY, AUTO_INCREMENT)
   - usuario_id (INT, FOREIGN KEY REFERENCES usuarios(id))
   - mensaje (TEXT, NOT NULL)
   - enviado (TIMESTAMP, DEFAULT CURRENT_TIMESTAMP)

RELACIONES ENTRE TABLAS:
------------------------
• usuarios → ventas (1:N) - Un usuario puede tener múltiples ventas
• usuarios → reseñas (1:N) - Un usuario puede dejar múltiples reseñas
• usuarios → chat (1:N) - Un usuario puede enviar múltiples mensajes
• productos → reseñas (1:N) - Un producto puede tener múltiples reseñas
• Las eliminaciones en cascada están configuradas para mantener la 
  integridad referencial

ÍNDICES:
--------
• Primary Keys en todas las tablas para identificación única
• Unique Key en email de usuarios para evitar duplicados
• Foreign Keys con índices automáticos para optimizar JOINs

7. MANUAL DE USUARIO
================================================================================

PARA CLIENTES:
--------------

1. REGISTRO:
   - Acceder a register.php
   - Completar nombre, email y contraseña
   - Hacer clic en "Registrarse"
   - Serás redirigido al login automáticamente

2. INICIO DE SESIÓN:
   - Acceder a login.php
   - Ingresar email y contraseña
   - Hacer clic en "Entrar"

3. COMPRAR PRODUCTOS:
   a) Navegar por la tienda (index.php):
      - Ver todos los productos disponibles
      - Usar filtros para buscar productos específicos
      - Leer reseñas de otros clientes
   
   b) Agregar al carrito:
      - Hacer clic en "Agregar al carrito" en el producto deseado
      - El producto se guarda en tu carrito de compras
   
   c) Ver carrito:
      - Acceder a carrito.php
      - Modificar cantidades si es necesario
      - Eliminar productos no deseados
      - Ver el total a pagar
   
   d) Checkout:
      - Hacer clic en "Proceder al pago"
      - Seleccionar método de envío
      - Seleccionar método de pago
      - Confirmar la compra
      - Descargar factura PDF automáticamente

4. SEGUIMIENTO DE PEDIDOS:
   - Acceder a perfil.php
   - Ver historial de compras
   - Hacer clic en "Rastrear" para ver el estado del pedido
   - Los estados van desde "Confirmado" hasta "Entregado"

5. DEJAR RESEÑAS:
   - En index.php, buscar el producto comprado
   - Completar el formulario de reseña
   - Escribir comentario y seleccionar calificación (1-5 estrellas)
   - Enviar reseña

6. CHAT DE SOPORTE:
   - Acceder a chat.php
   - Escribir mensaje en el área de texto
   - Hacer clic en "Enviar"
   - Esperar respuesta del administrador

PARA ADMINISTRADORES:
---------------------

1. ACCESO AL PANEL:
   - Iniciar sesión con credenciales de administrador
   - Serás redirigido automáticamente a admin.php

2. GESTIONAR INVENTARIO:
   a) Agregar producto:
      - Ir a inventario.php
      - Completar formulario con nombre, descripción, precio, stock, 
        categoría e imagen
      - Hacer clic en "Agregar Producto"
   
   b) Editar producto:
      - Hacer clic en "Editar" en el producto deseado
      - Modificar los campos necesarios
      - Hacer clic en "Actualizar Producto"
   
   c) Eliminar producto:
      - Hacer clic en "Eliminar"
      - Confirmar la acción en la alerta emergente

3. GESTIONAR VENTAS:
   - Acceder a ventas.php
   - Ver listado de todas las ventas realizadas
   - Cambiar el estado de un pedido usando el selector
   - Hacer clic en "Actualizar" para guardar el cambio

4. GESTIONAR USUARIOS:
   - Acceder a usuarios.php
   - Ver lista de todos los usuarios
   - Cambiar rol de usuario usando el selector desplegable
   - Eliminar usuarios (excepto administradores)

5. ATENDER CHAT:
   - Acceder a admin_chat.php
   - Leer mensajes de los clientes
   - Responder escribiendo en el área de texto
   - Hacer clic en "Enviar como Admin"

6. VER REPORTES:
   - Acceder a reportes.php
   - Ver métricas generales de ventas
   - Consultar ventas por fecha
   - Ver top clientes
   - Exportar reporte a PDF

7. ALERTAS DE STOCK:
   - En admin.php se muestran automáticamente alertas de productos
     con stock menor a 5 unidades
   - Reponer inventory accediendo a inventario.php

8. CONCLUSIONES
================================================================================

El desarrollo del sistema de comercio electrónico "Mi Tortuga" ha permitido 
implementar una plataforma funcional y completa que cubre las necesidades 
básicas de una tienda en línea. A lo largo del proyecto se lograron los 
siguientes aspectos destacados:

ASPECTOS POSITIVOS:
-------------------
1. INTERFAZ MODERNA: El diseño implementado con CSS3 ofrece una experiencia
   visual atractiva y profesional, con animaciones suaves y una paleta de
   colores elegante basada en tonos dorados y oscuros.

2. FUNCIONALIDAD COMPLETA: El sistema cuenta con todas las características
   esenciales de un e-commerce: catálogo, carrito, checkout, pagos, envíos,
   seguimiento y soporte.

3. SEGURIDAD: Se implementaron prácticas de seguridad básicas como:
   - Encriptación de contraseñas con password_hash()
   - Uso de sentencias preparadas para prevenir SQL Injection
   - Validación de sesiones para proteger rutas
   - Validación de archivos subidos

4. EXPERIENCIA DE USUARIO: Elementos como alertas SweetAlert2, confirmaciones
   de acciones, notificaciones de éxito y navegación intuitiva mejoran la
   experiencia del usuario.

5. GESTIÓN ADMINISTRATIVA: El panel de administración proporciona herramientas
   completas para gestionar todos los aspectos del negocio sin necesidad de
   conocimientos técnicos avanzados.

6. DOCUMENTACIÓN AUTOMÁTICA: La generación de facturas y reportes en PDF
   facilita la documentación de transacciones y el análisis del negocio.

ÁREAS DE MEJORA IDENTIFICADAS:
------------------------------
1. VALIDACIÓN DE FORMULARIOS: Se podría implementar validación más robusta
   del lado del cliente con JavaScript antes de enviar formularios.

2. PAGOS REALES: Actualmente los métodos de pago son simulados. Se podría
   integrar APIs reales como Stripe, PayPal o MercadoPago.

3. SISTEMA DE NOTIFICACIONES: Implementar notificaciones por email para
   confirmación de compras y cambios de estado de pedidos.

4. BÚSQUEDA AVANZADA: Mejorar el sistema de búsqueda con algoritmos más
   sofisticados y sugerencias automáticas.

5. OPTIMIZACIÓN DE IMÁGENES: Implementar compresión y redimensionamiento
   automático de imágenes subidas.

6. BACKUP AUTOMÁTICO: Crear un sistema de backup automático de la base de
   datos.

7. MULTI-IDIOMA: Agregar soporte para múltiples idiomas para alcanzar un
   público más amplio.

           ↓

ANEXO C: ESTADOS DEL CICLO DE VIDA DE UN PEDIDO
-----------------------------------------------

Estado 1: CONFIRMADO
- Compra realizada exitosamente
- Pago procesado
- Inventario actualizado

Estado 2: EN EMPAQUE
- Personal preparando el pedido
- Verificación de productos
- Empaque adecuado

Estado 3: EN TRANSPORTE
- Pedido entregado a paquetería
- En ruta hacia centro de distribución

Estado 4: EN CAMINO
